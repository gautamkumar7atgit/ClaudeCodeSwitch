## 1. Development Phases
# Development Phases

## Phase 1: Core Infrastructure (Week 1)
Build the foundational modules that all commands depend on.

**Deliverables:**
- `src/config.rs` — Constants and paths
- `src/keychain.rs` — Read/write Keychain via security CLI
- `src/profiles.rs` — Profile CRUD with atomic file writes
- `src/process.rs` — Find and kill claude processes
- Integration tests for each module

**Exit Criteria:**
- Can read/write Keychain programmatically
- Can create, read, update, delete profile JSON files
- Can find and terminate claude processes
- All integration tests pass

## Phase 2: CLI Commands (Week 2)
Implement all user-facing commands except daemon management.

**Deliverables:**
- `src/cli/mod.rs` — Clap command definitions
- `src/main.rs` — Entry point with dispatch
- `src/commands/add.rs`
- `src/commands/use_profile.rs`
- `src/commands/list.rs`
- `src/commands/remove.rs`
- `src/commands/status.rs`
- Integration tests for CLI commands

**Exit Criteria:**
- `ccswitch add <name>` captures credentials to profile
- `ccswitch use <name>` switches accounts in < 2 seconds
- `ccswitch list` shows all profiles with status
- `ccswitch remove <name>` deletes with confirmation
- `ccswitch status` shows current state
- All integration tests pass

## Phase 3: Daemon (Week 3)
Implement background sync daemon with launchd integration.

**Deliverables:**
- `src/daemon/mod.rs` — Poll loop with 30s interval
- `src/daemon/launchd.rs` — Plist generation and launchctl calls
- `src/commands/daemon.rs` — start/stop/status subcommands
- Log rotation logic
- Integration tests for daemon behavior

**Exit Criteria:**
- Daemon starts via launchd and stays alive
- Daemon detects Keychain changes and syncs to active profile
- Daemon logs to ~/.claude-switcher/daemon.log with rotation
- `ccswitch daemon start/stop/status` work correctly
- All integration tests pass

## Phase 4: Setup & Cleanup (Week 3, continued)
Implement first-run wizard and full uninstall.

**Deliverables:**
- `src/commands/init.rs` — First-run setup wizard
- `src/commands/uninstall.rs` — Full cleanup
- Integration tests for init and uninstall flows

**Exit Criteria:**
- `ccswitch init` creates directory structure, captures existing creds, starts daemon
- `ccswitch uninstall` stops daemon, removes all ccswitch files
- Clean macOS state after uninstall (Keychain untouched)
- All integration tests pass

## Phase 5: Release Engineering (Week 4)
Build universal binary and set up distribution.

**Deliverables:**
- GitHub Actions CI workflow for both architectures
- `lipo` step to create universal binary
- Release artifacts with SHA256 checksums
- Homebrew tap with formula

**Exit Criteria:**
- CI builds arm64 and x86_64 binaries
- Universal binary works on both Apple Silicon and Intel Macs
- `brew install` works from tap
- Manual install via curl works

## Phase 6: Documentation (Week 4, continued)
Write user-facing documentation.

**Deliverables:**
- README.md with install guide and quick start
- Command reference with all flags and examples
- Troubleshooting guide for common errors
- CHANGELOG.md

**Exit Criteria:**
- New user can install and set up ccswitch from README alone
- All commands documented with examples
- Common error scenarios covered in troubleshooting

## 5. Team Conventions
# Coding Conventions

## Rust Style
- **Edition:** 2021
- **Formatter:** `rustfmt` with default settings
- **Linter:** `clippy` with `#![warn(clippy::all)]`
- **Imports:** Group by `std`, external crates, internal modules; alphabetize within groups

## Error Handling
- Use `anyhow::Result<T>` for all fallible functions
- Add context with `.context("what failed")` at each error site
- Map shell command errors to meaningful messages
- Exit codes: 0 = success, 1 = usage, 2 = keychain, 3 = profile not found

## Naming
| Type | Convention | Example |
|------|------------|---------|
| Files | snake_case | `use_profile.rs` |
| Functions | snake_case | `read_keychain()` |
| Structs | PascalCase | `OAuthCredentials` |
| Constants | SCREAMING_SNAKE_CASE | `KEYCHAIN_SERVICE` |
| CLI args | kebab-case | `--force`, `--yes` |

## File Operations
- **Atomic writes:** Write to `.tmp` file, then `fs::rename()` to target
- **Permissions:** `chmod 600` on all profile JSON files (owner read/write only)
- **Path resolution:** Use `dirs::home_dir()`, never hardcode `/Users/`

## Process Management
- **SIGTERM first:** Send `SIGTERM`, wait 2 seconds
- **SIGKILL fallback:** If process still alive, send `SIGKILL`
- **PID discovery:** `pgrep -x claude` to find exact match

## Logging
- **CLI commands:** Silent by default, add `-v` flag for verbose mode
- **Daemon:** Always log to `~/.claude-switcher/daemon.log`
- **Format:** `[timestamp] [level] message` using `env_logger`
- **Rotation:** When `daemon.log` exceeds 1 MB, rename to `daemon.log.1`

## Testing
- **Integration tests after each milestone** (not TDD)
- **Test data:** Use `tests/fixtures/` for sample JSON files
- **Keychain mocking:** Test keychain functions against a test service name, not production
- **Cleanup:** Each test must clean up its test files and profiles
