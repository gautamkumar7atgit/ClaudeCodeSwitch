## 1. App Overview
# ccswitch — App Overview

## What It Is
ccswitch is a lightweight macOS CLI tool written in Rust that enables developers to seamlessly switch between multiple Claude Code subscription accounts from the terminal.

## Problem Solved
Claude Code authenticates via OAuth and stores credentials in the macOS Keychain under "Claude Code-credentials". There is no built-in multi-account support. Switching today requires running `/logout`, then `/login` with browser OAuth flow — slow, error-prone, and breaks developer flow.

## Solution
- Store each account's OAuth credentials as named JSON profiles on disk (`~/.claude-switcher/profiles/`)
- One command (`ccswitch use <name>`) writes correct credentials to Keychain, kills running Claude processes, and marks profile active
- Background daemon (launchd service) watches Keychain and syncs fresh tokens back to active profile after Claude Code performs silent OAuth refresh

## Core Goals
- Switch Claude Code accounts in < 2 seconds from terminal
- Keep all stored profiles up-to-date automatically after token refreshes
- Zero interaction required after initial profile setup
- Single static Rust binary — no runtime dependencies
- Full uninstall leaves macOS in clean state

## Non-Goals (MVP)
- macOS menu bar or status indicator
- Interactive TUI profile picker
- Keychain or encrypted storage for profiles
- Linux or Windows support
- Claude.ai browser session switching
- Multi-user or team credential sharing

## Target Users
| Persona | Use Case |
|---------|----------|
| Solo Dev | Switch between personal Claude Pro and client Claude Max |
| Freelancer | Manage 3–5 client Claude subscriptions |
| Agency Developer | Use shared team + personal accounts reliably |

## 2. Technology Stack
# Tech Stack

## Language & Toolchain
- **Language:** Rust (stable toolchain, 2021 edition)
- **Binary Output:** Single static binary, universal (arm64 + x86_64 via lipo)
- **Target OS:** macOS 12 Monterey and later

## Core Dependencies
| Crate | Purpose |
|-------|---------|
| `clap` (v4, derive) | CLI argument parsing — subcommands, flags, help generation |
| `serde` + `serde_json` | Serialize/deserialize profile JSON and daemon state |
| `anyhow` | Ergonomic error handling with context chains |
| `chrono` | Timestamps for sync tracking (`last_synced` field) |
| `nix` | SIGTERM / SIGKILL process management |
| `plist` | Write launchd .plist file for daemon installation |
| `dirs` | Resolve `$HOME` reliably across environments |
| `log` + `env_logger` | Structured logging in daemon, controlled via `RUST_LOG` |

## System Integration
- **Keychain Access:** Via macOS `security` CLI tool (no direct Security.framework FFI in MVP)
- **Process Management:** Via `nix` crate for signal handling
- **Daemon Management:** Via launchd with plist generation

## Performance Targets
- `ccswitch use <name>` completes in < 2 seconds end-to-end
- Daemon CPU: < 0.1% average (sleeps between 30s polls)

## 4. Module Breakdown
# Modules

## CLI Layer (`src/cli/`)
| File | Responsibility |
|------|----------------|
| `mod.rs` | Clap derive structs defining all subcommands and flags |

## Command Handlers (`src/commands/`)
| File | Command | Description |
|------|---------|-------------|
| `add.rs` | `ccswitch add <name>` | Capture current Keychain credentials → save as named profile |
| `use_profile.rs` | `ccswitch use <name>` | Kill claude processes → write profile to Keychain → set active |
| `list.rs` | `ccswitch list` | List all profiles with active marker and sync timestamps |
| `remove.rs` | `ccswitch remove <name>` | Delete profile from disk (with confirmation) |
| `status.rs` | `ccswitch status` | Show active profile, daemon status, last sync, credential match |
| `daemon.rs` | `ccswitch daemon {start\|stop\|status}` | Manage launchd daemon lifecycle |
| `init.rs` | `ccswitch init` | First-run wizard: create dirs, capture existing creds, start daemon |
| `uninstall.rs` | `ccswitch uninstall` | Full cleanup: stop daemon, delete ~/.claude-switcher/ |

## Core Modules (`src/`)
| File | Responsibility |
|------|----------------|
| `keychain.rs` | Read/write Keychain via `security` CLI; parse JSON credentials |
| `profiles.rs` | CRUD operations on `~/.claude-switcher/profiles/*.json`; atomic writes |
| `process.rs` | Find and kill running `claude` processes (SIGTERM → wait → SIGKILL) |
| `config.rs` | Constants: paths, `KEYCHAIN_SERVICE`, `POLL_INTERVAL`, exit codes |

## Daemon (`src/daemon/`)
| File | Responsibility |
|------|----------------|
| `mod.rs` | Main poll loop: sleep 30s → read Keychain → sync to active profile |
| `launchd.rs` | Generate plist XML; call `launchctl load/unload` |

## Entry Point
| File | Responsibility |
|------|----------------|
| `main.rs` | Parse args via clap, dispatch to command handlers |

## 5. Data Models
# Data Models

## Profile JSON Schema
Stored at `~/.claude-switcher/profiles/<name>.json` (chmod 600)

```json
{
  "claudeAiOauth": {
    "accessToken": "sk-ant-oat01-...",
    "refreshToken": "sk-ant-ort01-..."
  },
  "_meta": {
    "name": "work",
    "last_synced": "2026-04-07T10:30:00Z"
  }
}
```

| Field | Type | Description |
|-------|------|-------------|
| `claudeAiOauth.accessToken` | String | Short-lived OAuth bearer token (minutes to hours) |
| `claudeAiOauth.refreshToken` | String | Long-lived refresh token (weeks to months) |
| `_meta.name` | String | Profile name (matches filename without .json) |
| `_meta.last_synced` | ISO 8601 timestamp | When daemon last synced tokens from Keychain |

**Note:** When writing to Keychain, strip `_meta` — only `claudeAiOauth` object is written.

## Active Profile File
Stored at `~/.claude-switcher/active`

```
work
```

Plain text file containing the name of the currently active profile. No newline required but tolerated.

## Daemon Log
Stored at `~/.claude-switcher/daemon.log`

- Max size: 1 MB
- Rotation: `daemon.log` → `daemon.log.1` (keep one backup)
- Format: `[TIMESTAMP] [LEVEL] message`

```
[2026-04-07T10:30:00Z] [INFO] Daemon started, polling every 30s
[2026-04-07T10:30:30Z] [INFO] Keychain unchanged, skipping sync
[2026-04-07T10:31:00Z] [INFO] Token refresh detected, syncing to profile 'work'
[2026-04-07T10:31:00Z] [WARN] Failed to read Keychain: exit code 44
```

## Rust Structs

```rust
// src/profiles.rs
#[derive(Serialize, Deserialize, Debug, Clone)]
pub struct Profile {
    #[serde(rename = "claudeAiOauth")]
    pub oauth: OAuthCredentials,
    #[serde(rename = "_meta")]
    pub meta: ProfileMeta,
}

#[derive(Serialize, Deserialize, Debug, Clone)]
pub struct OAuthCredentials {
    #[serde(rename = "accessToken")]
    pub access_token: String,
    #[serde(rename = "refreshToken")]
    pub refresh_token: String,
}

#[derive(Serialize, Deserialize, Debug, Clone)]
pub struct ProfileMeta {
    pub name: String,
    pub last_synced: chrono::DateTime<chrono::Utc>,
}

// src/keychain.rs
#[derive(Serialize, Deserialize, Debug)]
pub struct KeychainCredentials {
    #[serde(rename = "claudeAiOauth")]
    pub oauth: OAuthCredentials,
}
```

## Exit Codes

| Code | Meaning |
|------|---------|
| 0 | Success |
| 1 | Usage error (bad args, missing subcommand) |
| 2 | Keychain error (read/write failed, no credentials found) |
| 3 | Profile not found |
